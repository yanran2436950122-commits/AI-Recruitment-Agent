# 2026-06

## 引入知识库幂等导入机制

状态：已完成


### 背景

知识库支持重复执行导入。

早期实现中，相同知识内容每次导入都会被重新写入向量库。

可能导致：

* 向量数量持续增长
* 检索结果重复
* 存储空间浪费

### 变更内容

知识块增加稳定标识：

```text
chunk_id
```

由以下信息生成：

```text
source
chunk_position
normalized_text
```

导入前执行重复检查。

### 涉及模块

```text
rag/vector_store.py
rag/ingest.py
```

### 结果

知识库导入具备幂等性。

重复导入不会产生重复向量记录。

## 增强知识库导入结果反馈

状态：已完成


### 背景

导入完成后仅显示导入成功。

无法判断：

* 是否新增内容
* 是否发生去重
* 当前知识库规模

### 变更内容

增加统计信息：

```text
files
chunks
inserted
skipped
total_documents
```

### 结果

能够直接判断导入效果与知识库状态。
